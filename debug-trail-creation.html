Debug Trail Creation

Steps to Test:

  1. Start the dev server: npm run dev
  2. Open browser console (F12)
  3. Navigate to Admin Dashboard from the navbar
  4. Click "Add Trail" in the Trails tab
  5. Fill minimum required fields:
    • Title: e.g., "Test Trail"
    • Type: Select any option
    • Main Image: Click on one of the available images
  6. Click "Create Trail"
  7. Check console for debug messages
  8. Go back to Home page
  9. Look for your new trail in the trail finder section
  10. Click "View Details" on your trail

What to Check:

  • ✅ Trail appears in the trail list with correct image
  • ✅ Trail shows "Trail details will be updated soon..." for missing description
  • ✅ Trail shows "TBD" for missing duration and difficulty
  • ✅ Clicking "View Details" navigates to detail page
  • ✅ Detail page shows all default content properly
  • ✅ Gallery section shows placeholder message or main image

Common Issues & Solutions:

Issue: Image not showing in trail list

  • Check browser console for 404 errors
  • Verify image path in localStorage data
  • Clear localStorage: localStorage.clear() in console

Issue: Detail page not loading

  • Check console for navigation errors
  • Verify trail ID is numeric
  • Check if dataManager.getTrailById() returns correct data

Issue: Gallery not working

  • Check if gallery array exists in trail data
  • Verify image paths are properly formatted
  • Test with placeholder.svg fallback

Debug Commands (Browser Console):

// Check all trails
console.table(JSON.parse(localStorage.getItem('lankaTrails_trails') || '[]'));

// Check specific trail by ID
const trails = JSON.parse(localStorage.getItem('lankaTrails_trails') || '[]');
console.log('Trail ID 8:', trails.find(t => t.id === 8));

// Clear all stored data
localStorage.clear();
location.reload();

// Test image loading
const img = new Image();
img.onload = () => console.log('Image loaded successfully');
img.onerror = () => console.log('Image failed to load');
img.src = '/assets/images/sigiriya.jpg';
